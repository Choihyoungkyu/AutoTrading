# 스펙 평가 피드백 — 주식 분석 시스템

> 평가일: 2026-06-24 · 기준: 3주(마감 7/15) · 팀 현실
> 대상 스펙: `spec.md` (주식 분석 시스템 MVP)

## 평가 전제 (수집된 맥락)

- **팀**: 3명, 모두 비슷한 수준 (Vue/Spring 풀스택 경험 있음 / Python·크롤링·AI 코딩은 첫 경험)
- **시간**: 취미 시간 병행 (구체 시간 미확정 — 최대 변수)
- **환경**: 자유로운 개발 환경 (크롤링·패키지 설치 제약 없음)
- **마감**: 7/15, 약 3주
- **핵심 우선순위**: 기업 분석 / 차트 분석이 메인 (뉴스 추천은 부차)

---

## 1. 종합평 — 🟡 (메인만 보면 🟢)

**메인인 기업/차트 분석은 3주에 충분히 완주 가능. 다만 5개 기능 전부를 욕심내면 빡빡하니, "뉴스 추천"은 가볍게 두는 게 안전.**

자동매매·키움·실거래가 V2로 빠지면서 가장 어려운 부분이 사라졌고, 핵심을 정형화된 작업(크롤링 + 지표 계산)으로 옮겨놓은 게 결정적으로 좋다.

## 2. 잘한 점

- **핵심을 잘 골랐다.** 기업/차트 분석은 공식이 명확(PER·RSI·MACD…)하고 `pandas`/`ta` 같은 검증된 라이브러리로 계산만 하면 됨. Python 첫 경험이어도 AI 코딩으로 넘기 쉬운 영역.
- **강점을 살리는 구성.** 대시보드를 익숙한 Vue로 두고 Python은 데이터·계산에만 써서 학습 곡선을 분산함.
- **막을 요인이 없음.** 자유로운 환경 → 크롤링·패키지 설치 OK. 치명적 제약 없음.

## 3. 부족한 점 (완주를 위협하는 것)

- **③ 뉴스 기반 추천이 유일한 발산 위험.** 메인이 아닌데도 5개 안에 같은 비중으로 들어가 있음. 시간을 여기 쏟다 메인이 약해질 수 있음 → 가점/축소 영역으로 명확히 내릴 것.
- **시간 총량이 여전히 안갯속.** "적당히"는 완주 계산이 안 됨. 취미 시간 × 3명이 실제 몇 시간인지에 따라, 5개 다 할지 / 메인 3개만 할지가 갈림.
- **크롤링 단일 의존.** 네이버 구조 변경·차단 시 멈춤 → 스펙에 적어둔 샘플/캐시 데이터 대체를 초반부터 깔아둘 것.
