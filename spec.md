# 주식 분석 시스템 - MVP Spec

## 1. 문제 정의

**핵심 문제**: 개인 투자자가 종목을 살지 말지 판단할 근거가 흩어져 있다.
- 재무 지표, 차트 지표, 시장 뉴스가 제각각 다른 곳에 있어 한눈에 보기 어렵다.
- 무엇을 보고 목표가·손절가를 잡아야 할지 기준이 없다.
- 한 종목을 여러 관점(기업·차트·시장)으로 **종합 분석한 리포트**를 자동으로 만들어, 의사결정을 돕는다.

> 이 시스템은 **분석·추천**까지만 한다. 주문·체결(자동매매)은 V2의 영역이다.

## 2. 타겟 사용자 (페르소나)

**개인 투자자**
- 직접 종목을 고르고 매매하지만, 분석에 쓸 시간·도구가 부족하다.
- 차트·재무·뉴스를 한 화면에서 보고, 종합 리포트로 빠르게 판단하고 싶다.

## 3. 핵심 기능 (V1 = 분석 시스템)

> **데이터 소스**: 국내증시는 **PYKRX**, 해외증시는 **yfinance** 라이브러리로 시세·재무 데이터를 가져온다 (크롤링 대신 안정적인 라이브러리 사용).
>
> ℹ️ **구현 메모**: 국내 **재무 지표**는 PYKRX `get_market_fundamental`을 쓰며, KRX 로그인이 필요하다(`.env`의 `KRX_ID`/`KRX_PW`). KRX는 별도재무 기준이라 부채비율은 별도로 yfinance를 사용한다. 경위는 `docs/adr/0005-재무지표-pykrx-복귀-krx로그인.md` 참조.

### ① 기업 분석 (재무 지표)
- 데이터 수집: 국내 **PYKRX**(재무는 KRX 로그인 필요) / 해외 **yfinance**
- 주요 지표 계산·표시: **PER, PBR, ROE, EPS, 부채비율, 배당수익률** 등
- 동종 업종/과거 대비 간단 비교 (저평가/고평가 시그널)

### ② 차트 분석 (기술적 지표)
- 과거 OHLCV(시세) 데이터 수집: 국내 **PYKRX** / 해외 **yfinance**
- 기술적 지표 계산: **이동평균선(MA), RSI, MACD, 볼린저밴드, 거래량**
- 매수/매도/관망 시그널을 룰 기반으로 산출

### ③ 종목 추천 (뉴스 기반) — *룰 기반으로 단순화*
- 뉴스 헤드라인 수집: 해외는 **yfinance(종목 뉴스)**, 국내는 **별도 크롤링/소스 필요** (라이브러리로 완전 대체 안 됨)
- **키워드 룰 기반** 긍정/부정 점수화 (예: 호재/악재 키워드 사전)
- 재무 + 차트 + 뉴스 점수를 합산해 **추천 종목 리스트** 산출
- ⚠️ 감성분석/ML이 아니라 **단순 키워드 룰**로 시작 (정확도보다 동작 우선)

### ④ 목표가 / 손절 라인 설정
- 분석 결과 기반으로 **목표 주가**와 **손절 라인** 자동 제안
- 룰 예: 목표가 = 현재가 × (1 + 기대수익률), 손절 = 현재가 × (1 − 허용손실률)

### ⑤ 종목 리포트 생성
- ①~④를 한 장으로 묶은 **종합 분석 리포트** 자동 생성
- 웹 대시보드에서 종목별 리포트 + 차트 시각화 확인

## 4. 제외 범위 (이번엔 안 함 → V2 이후)

- ❌ **자동매매 (주문·체결)** → **V2 핵심 기능**
- ❌ 실시간 시세 스트리밍 (V1은 과거/일별 데이터 기준)
- ❌ 키움 등 증권사 API 연동 (V2에서 자동매매와 함께)
- ❌ 머신러닝 기반 예측·감성분석 (V1은 룰 기반)
- ❌ 포트폴리오 최적화, 마진/옵션
- ❌ 모바일 앱 (웹 기반만)

> 📌 **범위 변경**: 기존 "국제 주식 제외" → **해외증시 포함** (yfinance 사용). 단, 시간이 빠듯하면 국내(PYKRX)를 먼저 완성하고 해외는 가점 영역으로 둔다.

## 5. 성공 기준

**필수 (모두 달성)**
- ✅ 종목(코드/이름)을 입력하면 **재무 지표 + 차트 지표**가 계산되어 표시된다
- ✅ 뉴스 크롤링 + 룰 점수로 **추천 종목 리스트**가 나온다
- ✅ 종목별 **목표가·손절 라인**이 자동 제안된다
- ✅ 위 내용을 묶은 **종합 리포트가 대시보드에 표시**된다
- ✅ 해커톤 제출 완료

**우수 (가점)**
- 다중 종목 비교 분석
- 차트 시각화 고도화 (지표 오버레이)
- 뉴스 점수화 정교화

## 6. 기술 스택

| 항목 | 선택 | 이유 |
|------|------|------|
| **데이터 수집** | **PYKRX**(국내) / **yfinance**(해외) | 안정적인 시세·재무 데이터 라이브러리 (크롤링 대체) — *국내 재무는 KRX 로그인 필요(`.env`), 부채비율은 yfinance. ADR 0005* |
| **뉴스 수집** | yfinance 뉴스(해외) + 국내 크롤링(BeautifulSoup/requests) | 국내 뉴스는 라이브러리 미지원 |
| **분석** | Python 3.9+ (pandas) | 금융 데이터·지표 계산 라이브러리 풍부 |
| **기술적 지표** | pandas / ta 라이브러리 | RSI·MACD 등 검증된 계산 |
| **백엔드 API** | Python (FastAPI/Flask) 또는 Spring | 팀이 Spring 경험 보유 → 택1 |
| **프론트엔드** | Vue | 팀의 강점(기존 경험) 활용 |
| **저장소** | SQLite / CSV | 수집·분석 결과 저장 |
| **환경** | 로컬 (Windows/Mac) | 개발 편의성 |

> 팀이 Python·크롤링은 처음이므로, **데이터 수집·지표 계산은 Python**, **대시보드는 익숙한 Vue**로 나눠 학습 곡선을 분산한다.

## 7. 개발 일정 (3주 / 3명 / 7월 15일 마감)

```
Week 1 (~7/1): 데이터 수집 + 지표 기반
  - 재무/시세 데이터 크롤링 (네이버 금융)
  - 기업 분석 지표 계산 (PER, PBR, ROE 등)
  - 차트 기술적 지표 계산 (MA, RSI, MACD, 볼린저)

Week 2 (~7/8): 분석 로직 + 추천 + 리포트
  - 뉴스 크롤링 + 키워드 룰 점수화
  - 종목 추천 로직 (재무+차트+뉴스 합산)
  - 목표가/손절 라인 제안 로직
  - 종합 리포트 데이터 구조 정의

Week 3 (~7/15): 대시보드 + 통합 + 발표
  - Vue 대시보드 (리포트 화면, 차트 시각화)
  - 전체 통합 테스트 + 버그 수정
  - 발표/시연 준비
```

### 역할 분담 (제안)
- **A** — 데이터 수집 담당 (재무·시세·뉴스 크롤링)
- **B** — 분석 엔진 담당 (기업/차트 지표 + 목표가/손절)
- **C** — 추천·리포트 + Vue 대시보드 담당
