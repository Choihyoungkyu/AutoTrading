# 주식 자동 매매 프로그램 (변동성 돌파 전략) - MVP Spec

## 1. 문제 정의

**핵심 문제**: 개인 투자자의 감정적 거래 제거
- 개인 투자자들이 시장 심리에 흔들려 비합리적 결정을 내림
- 일관된 알고리즘 규칙으로 자동거래하여 감정 개입 제거

## 2. 타겟 사용자 (페르소나)

**개인 투자자 (본인)**
- 기술 수준: 프로그래밍 가능
- 투자 경험: 있음
- 목표: 자동 매매 시스템 구축 및 실제 운영

## 3. 핵심 기능 (MVP)

### Phase 1: 데이터 & 백테스팅 (Day 1-10)
- [ ] 네이버 금융에서 삼성전자 과거 OHLCV 데이터 크롤링
- [ ] 키움 API 연동 (실시간 데이터 수신)
- [ ] **변동성 돌파 전략 로직 구현**
  - 계수: 0.8
  - 익절: +3%
  - 손절: -1%
  - 시간 손절: 15:20 (장 마감 10분 전 무조건 매도)
- [ ] 백테스팅 시스템 (과거 데이터로 전략 검증)
  - 수익률 계산
  - 거래 기록 저장
  - 승률, 최대손실, 평균 수익 등 분석

### Phase 2: 자동거래 (Day 11-20)
- [ ] 키움 API로 실시간 주가 수신
- [ ] 자동거래 로직 구현
  - 매매 신호 생성
  - 자동 주문 (매수/매도)
  - 익절/손절 자동 실행
  - 시간 손절 자동 실행
- [ ] 거래 기록 저장 (CSV 또는 SQLite)

### Phase 3: 대시보드 (Day 21-28)
- [ ] 기본 웹 대시보드 (JavaScript)
  - 현재 포지션 상태
  - 오늘의 수익/손실
  - 거래 기록 테이블
  - 백테스팅 결과 시각화

### Phase 4: 통합 & 배포 (Day 29-30)
- [ ] 전체 시스템 통합 테스트
- [ ] 버그 수정
- [ ] 실 운영 준비 (10만원 투자)

### Phase 4.5: 종목 분석 기능 (Day 29-30, 여유 있으면)
- [ ] 재무 데이터 API 연동 (네이버 금융 또는 키움)
- [ ] 다중 종목 재무 분석 (PER, PBR, ROE, 등)
- [ ] 유망 종목 필터링 (자동 스캔 기능)
- [ ] 분석 리포트 대시보드 표시

## 4. 제외 범위 (이번엔 안 함)

- ❌ 다중 종목 거래 (삼성전자만)
- ❌ 고급 포트폴리오 관리
- ❌ 고주파 거래 (초 단위 체계 불가)
- ❌ 마진/옵션 거래
- ❌ 국제 주식
- ❌ 모바일 앱 (웹 기반만)
- ❌ 리스크 관리 고급 기능 (VaR, 상관계수 등)

## 5. 성공 기준

**필수 (모두 달성해야 함)**
- ✅ 백테스팅 시스템이 정확하게 작동
- ✅ 키움 API 연동 완료
- ✅ 자동거래 로직이 실제로 작동 (주문 → 체결)
- ✅ 대시보드에서 포지션과 손익 확인 가능
- ✅ 해커톤 제출 완료

**우수 (추가 달성 시 가점)**
- 알고리즘 추가 구현
- 대시보드 UI 개선
- 추가 기술적 개선

## 6. 기술 스택

| 항목 | 선택 | 이유 |
|------|------|------|
| **백엔드** | Python 3.9+ | 금융 라이브러리 풍부, 키움 API 지원 |
| **자동거래 API** | 키움 KiwoomAPI | 국내 주식 자동거래 표준 |
| **데이터 수집** | 네이버 금융 크롤링 (BeautifulSoup/Selenium) | 과거 데이터 수집 |
| **프론트엔드** | JavaScript (Vanilla/React) | 대시보드 UI |
| **저장소** | SQLite / CSV | 거래 기록 저장 |
| **환경** | 로컬 (Windows/Mac) | 개발 편의성 |

## 8. 주의사항 & 위험 요소

| 위험 | 영향 | 대응 |
|------|------|------|
| **실금 투자 (5만원)** | 손실 가능성 100% | 코드 검증 후 매매, 버그 빠르게 대응 |
| **키움 API 학습 곡선** | 초반 3-4일 필요 | 공식 문서 + 커뮤니티 활용 |
| **30일 타이트 일정** | 과도한 기능 추가 시 실패 | 각 Phase별 일정 준수, 기능 추가 금지 |
| **백테스트 vs 실거래** | 차이 발생 가능성 | 종이 거래로 충분히 검증 후 진행 |
| **시장 변동성** | 예상 밖의 손실 | 손절 설정 (1%)으로 최악 상황 관리 |
