# 문제 추적 시스템 설정

## 개요

이 프로젝트는 **로컬 마크다운**으로 이슈를 관리합니다. 이슈는 저장소의 `.scratch/` 아래 개별 마크다운 파일로 저장됩니다.

## 구조

```
.scratch/
├── <기능명>/
│   ├── issue-001.md
│   ├── issue-002.md
│   └── ...
├── bugs/
│   ├── bug-001.md
│   └── ...
└── ...
```

## 워크플로우

1. **이슈 작성**: `.scratch/` 하위 적절한 디렉토리에 새로운 `.md` 파일 생성
2. **Triage**: YAML 프론트매터에서 라벨을 적용하며 진행
3. **종료**: 해결되면 파일 아카이브 또는 삭제

## 이슈 파일 예시

```markdown
---
title: "재무 지표 계산 구현"
labels: [needs-triage]
---

## 설명

PYKRX를 사용해 주어진 종목코드의 PER, PBR, ROE를 계산합니다.

## 완료 조건

- [ ] PER 정확하게 계산됨
- [ ] PBR 정확하게 계산됨
- [ ] ROE 정확하게 계산됨

## 담당자

팀원 A (데이터 수집)
```

## 외부 PR Triage

로컬 마크다운 추적 시스템이므로 외부 풀 리퀘스트는 자동으로 이슈로 triage되지 않습니다.
