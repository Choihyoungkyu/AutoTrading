# 도메인 문서

## 레이아웃

이 프로젝트는 **단일 컨텍스트** 레이아웃을 사용합니다:

- **CONTEXT.md** — 프로젝트의 목적, 타겟 사용자, 핵심 기능
- **docs/adr/** — 주요 기술 결정에 대한 아키텍처 결정 기록(ADR)

## 읽기 규칙

### CONTEXT.md 읽기

- 프로젝트의 목적, 타겟 사용자, 핵심 기능을 이해하기 위해 먼저 읽습니다
- 기능을 구현할 때 프로젝트 목표 정렬 확인을 위해 참고합니다
- 프로젝트 범위나 전략이 변경되면 업데이트합니다

### docs/adr/ 읽기

- 아키텍처 결정을 내릴 때 과거 결정 근거를 이해하기 위해 참고합니다
- 중요한 기술 결정을 할 때 새로운 ADR을 작성합니다
- ADR은 `docs/adr/NNNN-<결정제목>.md` 형식으로 저장합니다 (예: `0001-pykrx-yfinance-선택.md`)

## ADR 작성

이 템플릿을 사용합니다:

```markdown
# ADR NNNN: [결정 제목]

## 맥락

[결정이 필요한 문제 또는 상황]

## 결정

[내린 결정]

## 결과

[이 결정의 영향 — 긍정적, 부정적]
```

## 예상 ADR (작성 예정)

- `0001-국내-pykrx-해외-yfinance-선택.md`
- `0002-뉴스-감정분석-키워드룰-사용.md`
- `0003-v1-로컬-sqlite-저장소.md`
