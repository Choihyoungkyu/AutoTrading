# ML 매수신호 파이프라인 — 범위와 한계 (v1)

> 출처: `/grill-me` 세션에서 확정한 PRD(`.scratch/ml-signal-algorithm/prd.md`).
> 이 문서는 T07의 "한계 명시" 산출물이다. **이 신호는 투자 자문이 아니다.**

## 무엇인가

각 종목·각 시점에 대해 **"매수 / 매수 안 함"** 이진 신호와 매수 확률, 그 판단의
TreeSHAP 근거를 내는 지도학습(LightGBM) 파이프라인. 라벨은 "사서 -3% 손절 또는
5거래일 뒤 청산까지 실제로 돌렸을 때 **비용 제외 실현손익이 (+)였나**"이다(라벨=돈).

기존 규칙 기반 추천 엔진(`src/analyzers/recommendation_engine.py`, 재무·차트·뉴스
가중평균)과는 **별개의 병존 모듈**이며, 이를 대체하지 않는다.

## v1 실측 결론 (정직하게)

실데이터(코스피 대형주 12종목, 2021–2024, walk-forward 2폴드) 백테스트 결과
**성공 판정 = False**. 상위 신호 매매의 비용 제외 수익이 두 기준선(다 사는 전략·코스피
지수)을 요구 마진(거래비용 2배)으로 **꾸준히 이기지 못했다.** 즉 **차트 전용 v1은
대형주에서 유의미한 초과수익 근거가 없다.** 이 신호를 실거래에 쓸 근거는 아직 없다.

## 한계

- **차트 전용.** 재무·뉴스 미사용. 재무는 공시 시차(분기말+45일), 뉴스는 발행
  timestamp·감성모델 누수 검토가 필요해 v2로 미룸.
- **생존 편향 잔존.** 상폐·거래정지 종목 미포함 → 성능이 **낙관적으로** 나온다.
- **단일 지평.** 5거래일만. 1일/1개월 미지원.
- **수정주가 미처리(원본가).** 액면분할·증자 구간 가격 왜곡 가능성.
- **SHAP은 상관이지 인과가 아니다.** 개별 근거는 "모델의 자기 변명"이며 시장의
  이유가 아니다. 과신 금지.
- **KRX 로그인 의존.** point-in-time 유니버스·지수는 KRX 자격증명 필요(ADR 0007).
- **과최적화 위험.** threshold·top-k·피처를 백테스트에 맞춰 튜닝하면 가짜 성공이
  나온다. 시도 횟수(n_trials) 보고와 deflated Sharpe 보정 없이 나온 성공은 불신할 것.

## 신호를 올바르게 해석하는 법

- 지표는 전체 **정확도가 아니라** "가장 자신 있는 상위 신호의 비용 제외 수익"이다.
- 반드시 **두 기준선**(다 사는 전략·지수)과 비교하고, **여러 walk-forward 구간에서
  꾸준한지** 본다. 한 구간의 좋은 수익은 거의 항상 운이다.

## 라이브 API 미노출 (의도적 보류)

T07의 신호 엔드포인트는 **보류**한다. v1은 실측상 엣지가 없고 학습 모델도 영속화돼
있지 않다. 검증되지 않은 모델을 "매수 신호" API로 노출하면 투자 신호처럼 오인될 수
있어, 성공 판정을 통과하기 전에는 노출하지 않는다. 성능이 기준선을 꾸준히 이기게 된
뒤에 재검토한다.
