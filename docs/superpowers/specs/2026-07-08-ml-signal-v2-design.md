# ML 매수신호 파이프라인 v2 — 설계

**작성일**: 2026-07-08
**상태**: 승인됨 (브레인스토밍 확정)
**출처**: v1 결과(`docs/ml-signal-limitations.md`), ADR 0007/0008, `/grill-me` PRD

## 원칙

v1의 교훈을 계승한다: **돈 벌 근거가 없으면 정직하게 멈춘다.** 각 단계는 다음 단계의
*근거*를 만들며, 게이트를 통과하지 못하면 STOP·재검토한다. 게이트에서 STOP이 나오는
것은 실패가 아니라 정상적인 결과일 수 있다(v1이 그러했다).

## 배경

v1 실측 결론: **차트 전용·코스피 대형주 12종목·2폴드에서 성공 판정 = False**(두
기준선을 마진으로 꾸준히 이기지 못함). 다만 이 "엣지 없음"은 표본이 작아 통계적으로
약한 결론이다. v2는 이 결론 자체를 먼저 신뢰할 수 있게 만든 뒤, 무엇을 바꿔야
이길 수 있는지를 싸게 탐색한다.

## 로드맵 (D → C → B → A, 하드 게이트)

### Phase D — 검증 신뢰성 강화 *(항상 실행)*

"엣지 없음"을 믿을 수 있게 만든다.

- 진짜 point-in-time 유니버스 **100~200종목**으로 확대 (KRX 로그인 이미 작동, ADR 0007)
- walk-forward **5폴드+**(예: 2015~2024) + **락박스** 홀드아웃 (`splits.py` 지원)
- **수정주가 반영 OHLCV** — v1의 원본가 caveat 해소. 액면분할·증자 왜곡은 피처·라벨
  정확성 = 검증 신뢰성 문제이므로 D에서 처리한다.
- n_trials·deflated Sharpe·폴드 안정성 리포트 (`backtest.py` 경고 상수 활용)

**GATE D→C:** 측정이 폴드 전반에 안정적인가? 확대해도 여전히 엣지가 없으면 → C로
진행(신호/유니버스를 바꿔야 함이 입증됨). 만약 확대 표본에서 차트만으로 엣지가
나오면 → v1이 소표본 오판이었던 것이므로 별도 재평가한다.

### Phase C — 저비용 병렬 실험 *(항상 실행, D 통과 후)*

큰 투자 전에 "이 시장에서 뭐라도 이기나"를 싸게 확인한다. D의 강화된 하네스를
재사용하며, 각 arm을 **격리 측정**한다.

- **C1 (재무 min-viable):** 소수의 재무 피처 + **공시 시차(분기말+45일)**, 대형주 유지
- **C2 (중소형주 min-viable):** 차트 전용 그대로, 저순위 밴드/코스닥으로 유니버스만 교체

**GATE C→B/A:** 기준선을 이긴 arm만 다음으로. **C1 승 → A(재무) 우선. C2 승 →
B(유니버스) 우선. 둘 다 승 → B 먼저(요청 순서 D>C>B>A). 둘 다 실패 → STOP,
접근 전면 재검토.**

### Phase B — 유니버스 확대 + 생존편향 완화 *(C2 게이트 통과 시)*

- 중소형주 유니버스 풀 확대
- **상폐·거래정지 종목 point-in-time 포함** — v1 낙관 편향의 핵심 원인 제거. 시점별
  상폐 편입이 필요한 무거운 데이터 작업.

**GATE:** 락박스(out-of-sample)에서 엣지가 유지되는가?

### Phase A — 재무·뉴스 피처 확장 *(C1 게이트 통과 시)*

- 재무 피처 풀셋 (공시 시차 엄수)
- **뉴스 감성** — 발행 timestamp·감성모델 누수 통제 (누수 위험 최대)

**GATE:** 최종 락박스 + deflated Sharpe. 통과 시에만 노출/배포를 재검토한다(ADR
0008의 규칙엔진 대체 판단도 여기서).

## 이슈 분해 (12개)

| 단계 | 이슈 | 산출물 | 규모 | 라벨 |
|---|---|---|---|---|
| D | D1 수정주가 반영 OHLCV | 피처·라벨이 수정주가 사용 | M | ready-for-agent |
| D | D2 유니버스·기간·락박스 확대 | 100~200종목 × 5폴드+ | M | ready-for-agent |
| D | D3 신뢰성 리포트 + D→C 판정 | 안정성/deflated Sharpe, GO/STOP | S | ready-for-human |
| C | C1 재무 min-viable 실험 | 공시시차 재무 소수 + 격리 측정 | M | ready-for-agent |
| C | C2 중소형주 min-viable 실험 | 유니버스 교체 실험 | S | ready-for-agent |
| C | C3 실험 종합 + B/A 라우팅 판정 | 통과 arm 판정 | S | ready-for-human |
| B | B1 중소형주 유니버스 확대 | (게이트) 확대 유니버스 | M | needs-triage |
| B | B2 생존편향 완화 (상폐 포함) | (게이트) 시점별 상폐 편입 | L | needs-triage |
| B | B3 락박스 검증 + 판정 | (게이트) GO/STOP | S | ready-for-human |
| A | A1 재무 피처 풀셋 | (게이트) 공시시차 재무 전체 | M | needs-triage |
| A | A2 뉴스 감성 피처 | (게이트) 누수 통제 감성 | L | needs-triage |
| A | A3 최종 검증 + 배포 판정 | (게이트) deflated Sharpe·노출 결정 | M | ready-for-human |

라벨 규칙: 항상 실행되며 에이전트가 처리 가능한 실행 이슈 = `ready-for-agent`.
GO/STOP 판정 이슈(D3·C3·B3·A3) = `ready-for-human`(도메인 판단 필요). 게이트에
막혀 아직 착수 불가한 이슈(B1·B2·A1·A2) = `needs-triage`(해당 게이트 통과 시 승격).

## 일정 (가정: 1인/에이전트, 2026-07-08 착수, 게이트 조건부)

- **스프린트 1 (7/8~7/18):** Phase D 전체 (D1→D2→D3). **7/18 D→C 게이트**
- **스프린트 2 (7/21~7/31):** Phase C (C1·C2 병렬 → C3). **7/31 라우팅 게이트**
- **스프린트 3+ (8/4~):** 게이트 결과로 분기 — B(≈2.5주) 또는 A(≈3주). 둘 다면
  B→A 순, ~9월 중순 완료 예상
- 게이트에서 STOP이 나오면 그 지점에서 일정 종료·재설계

B·A는 조건부이므로 확정 일정이 아니라 "게이트 통과 시 예상 소요"다.

## 테스트 결정

- v1의 테스트 철학 유지: 외부 동작만 검증, 고정값·`patch` 목킹, `tests/test_ml_*.py`.
- 미래참조 통제 불변식은 v2에서도 최우선: 라벨=돈, T-불변 피처, embargo, 락박스 1회 개봉.
- 수정주가(D1)·공시시차(C1/A1)·뉴스 timestamp(A2)는 각각 **누수 회귀 테스트**를
  반드시 동반한다(과거 시점에 미래 정보가 새지 않는지).

## 범위 밖

- 실시간 스트리밍·주문 실행. 트랜스포머 계열 모델(현 규모·해석 요건에서 부적합, ADR 0008).
