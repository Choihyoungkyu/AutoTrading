# ADR 0006: 뉴스 수집 소스 다변화 (네이버 모바일 API + Google News RSS + yfinance)

**상태**: 채택됨

## 맥락

이슈 004(뉴스 기반 시장 심리 분석)는 데이터 소스로 **yfinance 뉴스 API**를 명시했다.
그러나 구현 중 다음이 확인되었다.

- yfinance는 이 환경에서 **국내 종목 데이터를 가져오지 못한다**(`005930.KS` → `No data
  found`). 이는 이슈 003 차트 분석이 실패했던 원인과 동일하며, 재무·차트는 이미
  pykrx/네이버로 전환된 상태다.
- 사용자 요구사항이 구체화되었다: **국내는 네이버, 해외는 yfinance, 공통으로 구글링**을
  포함하고, **최근 5일 이내** 뉴스만, **중복 내용은 1건만** 노출하며, **여러 종목에서
  재사용** 가능해야 한다.
- 네이버 종목뉴스 웹페이지(`news_news.naver`)는 익명 요청 시 빈 결과를 반환했다.
  반면 모바일 API(`m.stock.naver.com/api/news/stock/<code>`)는 **제목·언론사·발행시각·
  본문요약(body)**을 담은 깔끔한 JSON을 반환했다.
- raw 구글 검색은 차단·캡차 위험이 크지만, **Google News RSS**
  (`news.google.com/rss/search`)는 안정적이고 `pubDate`가 있어 5일 필터에 적합하며
  어떤 검색어(종목명)로도 재사용 가능하다.

## 결정

뉴스 수집을 **종목 유형에 따라 다중 소스로 다변화**한다.

- **국내(6자리 숫자 코드)**: 네이버 모바일 뉴스 API + Google News RSS(종목명 쿼리, `hl=ko`).
  종목명은 신규 `KRXCollector.get_name()`(pykrx `get_market_ticker_name`)으로 얻는다.
- **해외(티커)**: yfinance `Ticker.news` + Google News RSS(심볼 쿼리, `hl=en-US`).
- 각 소스는 개별 try/except로 감싸 **부분 실패를 허용**한다(한 소스가 죽어도 나머지로 진행).
- 수집 결과를 공통 스키마(`title/url/source/summary/published`)로 정규화한 뒤:
  - **최근 5일 필터**(`_within_days`, 순수 함수 — `now` 주입 가능해 테스트 결정적).
  - **중복 제거**(`_dedupe`): 제목 토큰 집합 **Jaccard 유사도 ≥ 0.6**이면 중복으로 보고
    먼저 등장한 것만 유지. 원본 매체(네이버/yfinance)를 구글보다 앞에 배치해 우선 보존.
  - 최신순 정렬 후 상한(`limit`) 적용.
- 감정 분석은 PRD대로 **키워드 룰 기반**(V1, ML은 V2). 헤드라인별 `[-1,1]` 점수 → 평균으로
  positive/neutral/negative 판정.
- 엔드포인트 `GET /analyze/<code>/news`는 **cache-first + 실패 시 캐시 폴백**
  (`source: "live" | "cache"`), `DataStore.save_news`/`load_news`(news_cache 테이블).
- 팝업(프론트) 본문: 외부 뉴스 사이트가 `X-Frame-Options`로 iframe 삽입을 차단하므로,
  네이버 `body` **요약 텍스트를 팝업에 표시**하고 원문은 새 탭 링크로 연결한다.

## 결과

**긍정**
- 국내 종목 뉴스가 실제로 수집된다(yfinance 국내 공백 해소). 신규 의존성 없음
  (`requests`/`bs4`/stdlib `xml.etree.ElementTree`·`email.utils`만 사용).
- 다중 소스로 커버리지↑, 부분 실패에 견고. 종목 코드로 재사용 가능.
- 후처리(필터·중복제거)가 순수 함수라 네트워크 없이 단위 테스트 가능.

**부정 / 트레이드오프**
- 이슈 004 문서의 "yfinance" 명세와 달라짐(본 ADR로 대체). 문서 정합성은 이 ADR로 유지.
- 구글 RSS 항목은 본문 요약이 없어(제목·링크뿐) 팝업 요약은 국내(네이버) 위주로 채워진다.
- 네이버 모바일 API·구글 RSS 응답 구조 변경 시 파싱이 깨질 수 있다(비공식 엔드포인트).
- 감정 사전이 단순 키워드라 문맥(부정어·반어)을 놓칠 수 있다(V2 개선 대상).
