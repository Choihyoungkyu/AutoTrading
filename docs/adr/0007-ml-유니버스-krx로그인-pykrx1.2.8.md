# ADR 0007: ML 유니버스·지수 조회를 위한 KRX 로그인(pykrx 1.2.8) 도입

**상태**: 채택됨

## 맥락

ML 매수신호 파이프라인(`.scratch/ml-signal-algorithm/prd.md`)의 유니버스 선정(T01,
`src/ml/universe.py`)은 **각 과거 시점(point-in-time)의 거래대금 상위 종목**이 필요하다.
성과 평가(T06)에는 **코스피 지수** 기준선이 필요하다.

- 이 두 데이터는 KRX 통계 엔드포인트(`get_market_ohlcv_by_ticker`(시장전체),
  `get_index_ohlcv`(지수))로만 point-in-time 조회가 된다.
- 그런데 이 엔드포인트는 **익명 요청을 거부한다**(KRX가 HTTP 400 본문 `LOGOUT` 반환).
  pykrx 1.0.47은 로그인 기능이 없어 빈 프레임 → 모호한 `KeyError`로 실패했다.
- 앱 본체(`krx_collector.py`)는 이미 시세·재무·거래량상위를 **네이버 금융**으로 이관해
  이 문제의 영향을 받지 않는다. 하지만 네이버 거래량상위는 **"현재" 순위만** 주므로
  과거 시점 유니버스(point-in-time)를 만들 수 없다 — ML 학습의 미래참조 통제 원칙과 충돌.
- `.env`에 유효한 `KRX_ID`/`KRX_PW`가 이미 존재했으나, 코드가 `load_dotenv()`를 전혀
  호출하지 않아 미사용 상태였다.

두 복구안(네이버로 교체 vs KRX 로그인)을 두고, **point-in-time 유니버스 원칙을 지키기
위해 KRX 로그인**을 택했다(사용자 결정).

## 결정

ML 파이프라인의 시장전체·지수 조회에 **pykrx 1.2.8 + KRX 로그인**을 사용한다.

- pykrx를 `1.0.47 → 1.2.8`로 업그레이드한다(`requirements.txt`). 1.2.8은 import 시점에
  `KRX_ID`/`KRX_PW`로 KRX에 로그인한다.
- pykrx는 **import 시점에 로그인**하므로, pykrx import **이전에** `dotenv.load_dotenv()`를
  호출해 자격증명을 환경에 먼저 로드한다. ML 진입점 `src/ml/universe.py` 상단에서 처리한다
  (ADR 0005에서 재무용으로 썼던 것과 동일한 패턴을 ML 유니버스용으로 재적용).
- `KRX_ID`/`KRX_PW`는 저장소 루트 `.env`에 두고 `.gitignore`로 커밋에서 제외한다.
- 앱 본체(`krx_collector.py`)는 **네이버 경로를 그대로 유지**한다. KRX 로그인은 ML
  파이프라인에서만 사용하며, 앱 동작을 강제로 바꾸지 않는다.

## 결과

**긍정**
- 진짜 point-in-time 유니버스(과거 각 시점 거래대금 상위)와 실제 코스피 지수 기준선을
  얻어, 미래참조 통제 원칙을 지킨 채 학습·백테스트가 가능하다.
- 시장전체(953종목)·지수 엔드포인트가 실제로 부활함을 검증했다.

**부정 / 트레이드오프**
- **자격증명 필요**: `.env`의 `KRX_ID`/`KRX_PW`가 없으면 시장전체·지수 조회가 빈 값이 되고
  유니버스 선정이 실패한다(`get_universe`는 빈 리스트 반환).
- **로그인 세션 1시간 만료**: 배치성 실행엔 무리 없으나, 장시간 서비스에는 재로그인 처리가
  필요할 수 있다.
- **pykrx 1.2.8 업그레이드 파급**: 의존 트리로 numpy가 2.4.6으로 상향됐다(전체 테스트
  회귀 없음 확인). 또한 pykrx import 시 자격증명이 로드돼 있지 않으면 콘솔에 로그인 실패
  경고가 출력된다(앱은 네이버 경로라 기능엔 영향 없음).
- **수정주가 미처리(원본가)**: 유니버스는 거래대금 순위만 쓰므로 영향이 제한적이나, 가격을
  직접 쓰는 하위 단계(피처·라벨)는 액면분할·증자 구간에서 왜곡 가능성이 있다.

**보안**
- `.env`는 `.gitignore`에 포함되어 커밋되지 않는다. 자격증명은 data.krx.co.kr로만 전송된다.
