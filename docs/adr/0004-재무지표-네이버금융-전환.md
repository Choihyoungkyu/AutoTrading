# ADR 0004: 재무 지표 데이터 소스를 PYKRX(KRX)에서 네이버 금융으로 전환

> ⚠️ **대체됨(Superseded)**: 이 결정은 [ADR 0005](0005-재무지표-pykrx-복귀-krx로그인.md)로 번복되었다.
> KRX 로그인 자격증명을 확보하여 재무 지표는 다시 pykrx로 복귀했다. 아래는 당시 기록으로 보존한다.

## 맥락

`KRXCollector.get_market_cap()`은 본래 `pykrx.stock.get_market_fundamental()`로
한국거래소(KRX)에서 PER/PBR/EPS/BPS/배당수익률을 가져왔다. 그런데 재무 분석
엔드포인트(`/analyze/<code>/financial`)가 "No data found"를 반환하는 문제가 있었다.

원인을 직접 검증한 결과:

- KRX 재무 엔드포인트(`https://data.krx.co.kr/comm/bldAttendant/getJsonData.cmd`,
  `bld=...MDCSTAT03502`)는 **익명 요청에 `400 LOGOUT`을 반환**한다. 익명
  세션 쿠키(JSESSIONID)를 붙여도 동일하게 거부된다 → **로그인이 필요**하다.
- 설치된 pykrx에는 `KRX_ID`/`KRX_PW` 환경 변수로 로그인하는 코드가 추가돼
  있으나, 자격증명이 없으면 익명 폴백으로 빈 응답(`Expecting value`)을 받는다.
- 반면 **시세(OHLCV)는 영향이 없다**. `pykrx`의 `get_market_ohlcv`는 내부적으로
  네이버 금융에서 데이터를 가져오기(`stock_api.py`의 `adjusted` 분기) 때문이다.

즉, 재무 데이터만 KRX 로그인 뒤에 막혀 있었다. 로그인은 별도 KRX 계정(소셜
로그인만으로는 ID/PW 폼 로그인 불가)과 크레덴셜 관리가 필요해, 해커톤 범위에
부담이 된다.

이 결정은 ADR 0001(국내 PYKRX / 해외 yfinance 선택, 작성 예정)의 "국내 재무는
PYKRX" 부분을 부분 수정한다.

## 결정

`get_market_cap()`을 **네이버 금융 종목 페이지 스크래핑**으로 전환한다.

- 출처: `https://finance.naver.com/item/main.naver?code=<code>`
- 파싱: `requests` + `BeautifulSoup(html.parser)` — `div.aside_invest_info
  table.per_table`에서 PER/EPS/PBR/BPS/배당수익률 추출. 추가 의존성 없음
  (`requests`, `beautifulsoup4`는 이미 `requirements.txt`에 존재, lxml 불필요).
- 메서드 시그니처와 반환 dict(`per/pbr/eps/bps/dividend_yield`)를 그대로
  유지하여 호출자(`FinancialAnalyzer`, `routes.py`)는 변경하지 않는다.
- 호출 빈도를 줄이기 위해 재무 분석 결과를 **일일 캐싱**한다
  (`DataStore.save_financial()/load_financial()`, `financial_cache` 테이블;
  `/analyze/<code>/financial`에서 read-through).
- 시세(OHLCV)는 기존대로 PYKRX 경로를 유지한다.

## 결과

**긍정**
- KRX 로그인/크레덴셜 없이 재무 분석이 복구된다. `KRX_ID`/`KRX_PW`/`.env` 불필요.
- 추가 라이브러리 의존성이 없다.
- 일일 캐싱으로 외부 요청과 차단 위험을 줄인다.

**부정 / 트레이드오프**
- spec의 "크롤링 대신 안정적인 라이브러리" 원칙에서 **재무 부분만 크롤링으로
  회귀**한다(시세·해외는 라이브러리 유지).
- 네이버 금융 HTML 구조가 바뀌면 파싱이 깨질 수 있다(셀렉터 의존).
- 네이버는 **최신값만** 제공한다. `get_market_cap(code, date)`의 `date` 인자는
  호환성을 위해 남겨두지만 무시되며, 과거 특정일 재무 조회는 불가하다(현재
  호출자는 date를 넘기지 않아 동작에는 차이 없음).

관련 커밋: `65d4800` (feat: 재무 데이터 KRX 로그인 의존 제거 및 일일 캐싱 추가).
