# 주식 분석 시스템 MVP PRD

## 핵심 네러티브

월급의 30%를 주식에 투자하는 직장인 A는 "괜찮은 기업이라고 생각하면" 매수한다. 하지만 투자 직후 주가가 하락하면 불안해져 손실을 감수하고 빠져나가고, 반대로 쉽게 오른 종목은 "놓칠까봐" 과도한 자금을 투자하다가 급락에 휘말린다. 진짜 문제는 정보 부족이 아니었다. **의사결정이 늦고**, 시장의 변동에 시달려 **기회를 놓친다**는 것이었다. 네이버 금융, 차트 앱, 뉴스까지 본들 "이 정보가 무엇을 의미하는가" 스스로 판단하는 데는 시간이 걸린다. 그 사이 기회는 떠나간다.

기술 장벽이 낮아진 지금, 생성AI와 공개 데이터 API를 활용하면 개인도 **5분 만에** 재무·차트·뉴스를 종합 분석하고 객관적 신호를 얻을 수 있다. 우리의 베팅은 간단하다: 투자의 성공은 정보의 많음이 아니라 **빠른 판단**과 **감정 제거**에 있다는 것이다. 초보자도 접근할 수 있는 도구가 있으면, 시장 변화에 빠르게 반응하고 패닉 없이 객관적 신호로 판단할 수 있다.

---

## Problem Statement

개인 투자자들(특히 정보 분석에 자신 없는 직장인)이 종목 투자 결정을 내릴 때, 필요한 정보가 여러 곳에 흩어져 있습니다:
- 재무 지표 데이터 (네이버 금융, 한국거래소 등)
- 기술적 지표 및 차트 (자신의 차트 분석 도구)
- 최신 뉴스 (뉴스 포털, SNS)
- 업종 평균 대비 평가 정보

이로 인해:
1. 종목을 판단하기까지 여러 웹사이트를 돌아다니느라 **의사결정이 늦어지고**
2. 정보 취합 중 시장 기회를 **놓친다**
3. 스스로 판단할 명확한 기준이 없어 시장 변동에 휘말려 **감정적 판단(패닉바이/셀)에 빠지기 쉬워진다**

## Solution: 빠른 판단과 감정 제거를 위한 통합 분석 도구

**주식 분석 시스템**은 종목 코드만 입력하면, **5분 이내에** 다음을 자동으로 제공합니다:

### 핵심 가치
- **빠른 의사결정**: 점심시간 5분, 출근길 2분. 정보 취합이 아니라 판단만 한다.
- **감정 제거**: 규칙 기반 신호. "내 생각"이 아니라 데이터의 말을 따른다.
- **초보자 접근성**: 재무제표, 차트 분석 경험 없어도 된다. 시스템이 해석해서 전달한다.

### 제공 정보
1. **재무 분석**: PER, PBR, ROE, EPS, 부채비율, 배당수익률 등의 지표와 동종업계 평균 비교 → "이 기업은 지금 저평가인가?"
2. **기술적 분석**: 이동평균선, RSI, MACD, 볼린저밴드를 기반으로 한 매수/매도 시그널 → "차트상 진입 신호는?"
3. **뉴스 기반 평가**: 최신 뉴스 헤드라인의 긍정/부정 키워드 분석 → "시장 심리는 긍정적인가?"
4. **종합 추천 등급**: 재무 + 차트 + 뉴스 점수를 합산한 Buy/Hold/Sell 신호 → "지금 사야 하나?"
5. **진입/청산 가이드**: 목표가와 손절 라인의 자동 제안 → "목표는 얼마, 손절은 어디?"

**모든 정보를 한 장의 종합 리포트로 웹 대시보드에 표시**하여, 직장인도 빠르고 객관적으로 판단할 수 있도록 합니다.

## User Stories

1. As an 개인 투자자, I want to 종목코드를 입력하면 재무 지표를 자동으로 계산 및 표시되기를, so that 네이버 금융 등 여러 사이트를 돌아다니지 않을 수 있습니다

2. As an 개인 투자자, I want to 현재가 대비 PER, PBR이 저평가/고평가인지 한눈에 보기를, so that 기업가치를 빠르게 판단할 수 있습니다

3. As an 개인 투자자, I want to 동종업계 평균 지표와 비교된 결과를 보기를, so that 상대적 평가를 할 수 있습니다

4. As an 개인 투자자, I want to 과거 OHLCV 데이터 기반 기술적 지표(MA, RSI, MACD)가 계산되기를, so that 차트 기반 진입 신호를 얻을 수 있습니다

5. As an 개인 투자자, I want to 기술적 지표들이 현재 매수/매도/관망 신호를 제시하기를, so that 진입 타이밍을 객관적으로 판단할 수 있습니다

6. As an 개인 투자자, I want to 최신 뉴스 헤드라인을 자동 수집하고 호재/악재 키워드로 점수화하기를, so that 시장 센티먼트를 파악할 수 있습니다

7. As an 개인 투자자, I want to 재무 점수 + 차트 점수 + 뉴스 점수가 합산된 종합 추천 등급을 보기를, so that 여러 관점을 균형있게 고려한 투자 결정을 할 수 있습니다

8. As an 개인 투자자, I want to 현재가 기반 목표 주가가 자동으로 제안되기를, so that 수익 목표를 설정할 수 있습니다

9. As an 개인 투자자, I want to 손절 라인이 자동으로 제안되기를, so that 손실 한정 전략을 미리 계획할 수 있습니다

10. As an 개인 투자자, I want to 위의 모든 정보를 한 장의 종합 리포트로 보기를, so that 투자 결정에 필요한 모든 정보를 한눈에 파악할 수 있습니다

11. As an 개인 투자자, I want to 종목 리포트를 웹 대시보드에서 조회하고 차트를 시각화해서 보기를, so that 집에서 편하게 분석할 수 있습니다

12. As an 개인 투자자, I want to 국내 주식뿐 아니라 해외 주식도 분석하기를, so that 글로벌 투자 기회를 발굴할 수 있습니다 (우수 기능)

13. As a 팀 개발자, I want to 데이터 수집 모듈이 PYKRX(국내)와 yfinance(해외)에서 안정적으로 데이터를 가져오기를, so that 크롤링 차단 위험을 최소화할 수 있습니다

14. As a 팀 개발자, I want to 각 분석 모듈(재무, 차트, 뉴스)이 독립적으로 테스트 가능하기를, so that 역할별 개발이 병렬로 진행될 수 있습니다

15. As a 팀 개발자, I want to 뉴스 크롤링이 차단되더라도 시스템이 작동하기를, so that 뉴스 수집 실패가 전체 시스템 장애로 이어지지 않습니다

## Implementation Decisions

### 1. 데이터 수집 전략

**결정**: 국내 시세·재무는 PYKRX 라이브러리, 해외는 yfinance 라이브러리 사용. 뉴스는 yfinance 뉴스 API(해외) + 국내 크롤링 병행.

> ℹ️ **구현 메모**: 국내 **재무 지표**는 PYKRX `get_market_fundamental`로 수집하며 KRX 로그인이 필요하다(`.env`의 `KRX_ID`/`KRX_PW`). 부채비율은 KRX 미제공이라 yfinance를 사용한다. (네이버 전환을 검토했다가 KRX 로그인 확보로 복귀 — 경위는 `docs/adr/0005-재무지표-pykrx-복귀-krx로그인.md`, 이전 결정은 ADR 0004.)

**근거**: 
- 라이브러리 기반 수집 → 크롤링 차단 위험 최소화 → **지속 가능한 서비스**
- 안정적인 데이터 수집 → 개발 시간 절약 → **빠른 의사결정의 기반**
- 개인도 운영할 수 있는 기술 스택 → **기술 장벽 낮춤**

### 2. 분석 엔진 아키텍처

**결정**: Python 기반 분석 엔진 (pandas, ta 라이브러리 사용)으로 모듈화:
- `FinancialAnalyzer` — 재무 지표 계산
- `ChartAnalyzer` — 기술적 지표 계산 및 신호 생성
- `NewsAnalyzer` — 뉴스 크롤링 및 키워드 기반 점수화
- `RecommendationEngine` — 종합 점수 및 추천 로직
- `PriceTargetCalculator` — 목표가/손절 제안

**근거**: 각 모듈이 독립적으로 테스트·개발 가능 → 3명 팀의 병렬 작업 용이

### 3. 감정분석 방식

**결정**: V1은 키워드 룰 기반 감정분석 (사전 기반 긍정/부정 점수화)

**근거**: 
- 머신러닝 학습 데이터 준비 시간 절약
- 빠른 프로토타이핑 및 검증 가능
- 정교화는 V2 우수 기능으로 미룸

### 4. 저장소 및 캐싱

**결정**: SQLite 또는 CSV로 수집·분석 결과 저장. 뉴스 크롤링 실패 시 캐시 데이터 사용.

**근거**: 로컬 환경에서 간단하고 빠른 구현 / 뉴스 수집 실패 시에도 시스템 작동

### 5. 백엔드 API

**결정**: FastAPI 또는 Flask로 RESTful API 구현. Primary seam은 `/analyze/<stock_code>` 엔드포인트.

**근거**: 
- 프론트엔드(Vue)와 명확한 계약 → 통합 테스트 용이
- 데이터 수집·분석 로직은 백엔드에서만 실행 → 프론트엔드 부담 감소

### 6. 프론트엔드

**결정**: Vue 기반 대시보드. 종목별 리포트 카드 + 차트 시각화 (Chart.js 등).

**근거**: 팀의 기존 Vue 경험 활용 / 빠른 UI 개발 가능

### 7. 데이터 흐름

```
사용자 입력 (종목코드)
    ↓
백엔드 API (/analyze/<stock_code>)
    ↓
[병렬 실행]
├─ FinancialAnalyzer → 재무 지표 계산
├─ ChartAnalyzer → 차트 지표 계산
├─ NewsAnalyzer → 뉴스 점수 계산
└─ [위 결과 취합]
    ↓
RecommendationEngine → 종합 추천 계산
    ↓
PriceTargetCalculator → 목표가/손절 계산
    ↓
종합 분석 결과 (JSON)
    ↓
Vue 대시보드 → 시각화 및 표시
```

## Testing Decisions

### 테스트 철학

좋은 테스트는:
- **외부 동작만 검증** — 구현 세부사항이 아니라 입력에 대한 출력이 맞는지만 확인
- **독립적으로 실행 가능** — 다른 테스트나 외부 서비스 의존도 없음
- **빠르고 결정적** — 네트워크 또는 시간대에 따라 결과가 변하지 않음

### Primary Seam 테스트

**엔드포인트**: `GET /analyze/<stock_code>`
- **입력**: 종목 코드 (예: "005930" 삼성전자)
- **출력**: 다음을 포함한 JSON 응답
  ```json
  {
    "stock_code": "005930",
    "stock_name": "삼성전자",
    "financial": { "per": 8.5, "pbr": 1.2, "roe": 12.5, ... },
    "chart": { "signal": "buy", "ma_20": 70000, "rsi": 35, ... },
    "news": { "score": 0.6, "headlines": [...] },
    "recommendation": { "grade": "buy", "score": 0.75, ... },
    "price_target": { "target": 75000, "stop_loss": 60000 }
  }
  ```
- **테스트 전략**:
  - 정상 종목: 예상된 구조의 JSON 반환 확인
  - 존재하지 않는 종목: 적절한 오류 응답 확인
  - 데이터 수집 실패: 캐시 데이터로 폴백 확인

### Secondary Seam 테스트 (모듈별)

**FinancialAnalyzer.calculate(stock_code)**
- 테스트 대상: PYKRX/yfinance에서 가져온 데이터로 PER, PBR, ROE, EPS 등이 정확하게 계산되는가
- 선행 사례: 금융 라이브러리 단위 테스트 (ta 라이브러리의 지표 테스트 참고)
- Mock: 데이터 수집 부분은 고정 데이터셋으로 mock

**ChartAnalyzer.calculate(ohlcv_data)**
- 테스트 대상: 과거 OHLCV 데이터로부터 MA, RSI, MACD, 볼린저밴드 계산 및 신호 생성
- 선행 사례: ta 라이브러리의 지표 검증 테스트
- Mock: 고정 시계열 데이터 (pandas DataFrame)

**NewsAnalyzer.calculate(headlines)**
- 테스트 대상: 뉴스 헤드라인 목록에 대해 키워드 룰 기반 점수가 정확하게 계산되는가
- 선행 사례: 감정분석 단위 테스트
- Mock: 테스트용 고정 헤드라인 리스트

**RecommendationEngine.generate(fin_score, chart_score, news_score)**
- 테스트 대상: 3개 점수를 합산하여 추천 등급(buy/hold/sell)이 올바르게 결정되는가
- 선행 사례: 규칙 기반 엔진 테스트 (상태 머신)
- Mock: 고정 점수 입력

**PriceTargetCalculator.suggest(current_price, expected_return, max_loss)**
- 테스트 대상: 현재가와 기대수익률·허용손실률로부터 목표가·손절가가 정확하게 계산되는가
- 선행 사례: 금융 계산 공식 테스트
- Mock: 고정 가격 데이터

### 테스트 모듈

- **Backend API**: pytest 기반 엔드투엔드 테스트
- **분석 엔진**: pytest 기반 단위 테스트 (mock 데이터 사용)
- **Frontend**: Vue 컴포넌트 테스트는 선택사항 (시간이 부족하면 수동 검증)

### 테스트 환경

- **테스트 데이터**: 고정 OHLCV, 재무 데이터, 뉴스 헤드라인 파일
- **Mock 대상**: 외부 API (PYKRX, yfinance, 뉴스 크롤링)
- **실행 환경**: GitHub Actions 또는 로컬 (pytest)

## Out of Scope

❌ **자동매매 (주문·체결)** → V2 핵심 기능
- 증권사 API (키움증권 등) 연동은 V2에서만 진행
- 이 PRD는 분석 및 추천 리포트까지만 포함

❌ **실시간 시세 스트리밍**
- V1은 과거·일별 데이터 기준
- 실시간 업데이트는 V2 성능 최적화 단계에서 고려

❌ **머신러닝 기반 예측 및 고급 감정분석**
- V1은 규칙 기반 감정분석 (키워드 사전)
- NLP/ML 모델 학습은 V2 우수 기능

❌ **포트폴리오 최적화, 마진/옵션**
- 단일 종목 분석만 포함
- 다중 종목 조합 전략은 V2 이후

❌ **모바일 앱**
- 웹 기반 대시보드만 구현

❌ **실시간 알림 (Slack, Email, SMS)**
- V2 기능

### 성공 기준 (MVP)

**필수 (모두 달성) — "직장인 A가 점심시간 5분에 판단을 내릴 수 있는가"**
- ✅ 종목 코드 입력 → **5초 이내** 재무 지표 + 차트 지표 계산·표시
- ✅ 뉴스 크롤링 + 감정 점수 → 시장 심리 반영된 추천 신호
- ✅ 목표가·손절 라인 자동 제안 (투자 계획의 명확화)
- ✅ **한 화면의 종합 리포트** — 여러 탭을 누를 필요 없음
- ✅ 객관적 신호(Buy/Hold/Sell) 제시 — 자기 판단의 여지 최소화
- ✅ 해커톤 제출 완료 및 시연

**우수 (가점) — "더 빨리, 더 정확하게"**
- 다중 종목 비교 분석 (오늘은 어느 5종목을 봐야 하나?)
- 차트 시각화 고도화 (지표 오버레이로 신호 명시화)
- 뉴스 감정분석 정교화 (호재/악재 구분 정확도 ↑)
- 해외 주식 지원 (글로벌 기회 발굴)

## 네러티브 추출 및 검증

### 핵심 네러티브 구성 요소

| 요소 | 내용 | 출처 |
|------|------|------|
| **대상** | 월급의 30%를 주식에 투자하지만 정보 분석에 자신 없고, 시장 상황에 흔들려 패닉바이/패닉셀을 반복하는 직장인 | 명시 |
| **진짜 문제** | 의사결정이 느려서 시장 기회를 놓치고, 감정적 판단에 빠진다 | 명시 |
| **맥락·타이밍** | 기술 장벽이 낮아져 개인도 AI/API를 활용한 투자 분석 자동화가 가능해졌다 | 명시 |
| **핵심 관점·베팅** | 투자 성공은 정보의 양이 아니라 빠른 판단과 감정 제거에 있다. 초보자도 접근 가능한 도구가 있으면 객관적으로 판단할 수 있다 | 명시 + 추론 |
| **해결 방향** | 종목 코드만 입력하면 재무/차트/뉴스를 자동 분석하고 객관적 신호(Buy/Hold/Sell)를 제시하는 웹 도구 | 명시 |
| **성공의 모습** | 직장인이 점심시간 5분 만에 투자 결정을 내리고, 시장 변동에 흔들리지 않고, 기회를 놓치지 않는다 | 추론 |

### 진단

**강한 지점**
- ✅ 구체적 대상: "월급 30% 투자 + 정보 분석 미숙 + 패닉 거래"라는 페르소나가 명확함
- ✅ 근본 문제와의 연결: "정보 부족"이 아니라 "의사결정 지연 + 감정적 판단"이라는 근본 문제가 드러남
- ✅ 기술과 비즈니스의 정렬: "기술 장벽 낮음"이라는 시대적 기회가 네러티브와 연결됨
- ✅ 측정 가능한 성공: "5분 내 판단" → 명확한 성공 기준

**검증 완료**
- 네러티브가 Product Scope와 일치: MVP의 모든 기능이 "빠른 판단 + 감정 제거"라는 핵심 목표를 향함
- User Stories와의 정렬: 12개 스토리가 모두 "신호 제시 → 판단 시간 단축"으로 귀결됨
