# T06: 백테스트 평가기 (S4) + 성공 판정

## What to build

신호를 확률순 정렬해 상위 신호만 매매하는 백테스트를 돌리고, 두 기준선 대비 비용 제외 초과 성과로 성공/실패를 판정한다.

**End-to-end 동작:**
```
backtest(signals+probs, prices, cost, top_k) ->
  {누적수익, 바보전략_대비, 지수_대비, 구간별_성과, 성공여부}
```

## Acceptance criteria

- [ ] 신호를 확률 내림차순 정렬 → 상위 top-k만 매매 (전체 정확도 아님)
- [ ] 성과 = 비용 제외 누적수익
- [ ] 기준선 2개 비교: ① 바보 전략(다 사서 5일 청산) ② 코스피 지수 보유
- [ ] **성공 정의:** walk-forward 구간 대다수(예: 3개 중 2개↑)에서 두 기준선을 **왕복거래비용 2배 이상 마진**으로 상회
- [ ] 한 구간만 좋으면 운으로 간주 → 실패
- [ ] **과최적화 경고 반영(조사 리포트):** 시도한 시행 횟수(N) 보고, deflated Sharpe 보정 고려 — 튜닝 반복이 성과를 부풀렸는지 명시
- [ ] 결과 보고에 **생존 편향으로 성능이 낙관적**임을 명시
- [ ] lockbox 구간은 최종 1회만 개봉

## Seam (테스트 필수)

- 고정 신호+가격으로 top-k 매매 수익이 비용 차감 후 정확히 계산
- 바보 전략·지수 기준선 수익 계산 정확성
- 성공 판정 로직(구간 대다수 + 2배 마진)이 경계에서 옳게 True/False
