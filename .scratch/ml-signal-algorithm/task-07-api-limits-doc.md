# T07: (선택) API 노출 + 한계 명시 문서

## What to build

신규 ML 신호를 선택적으로 API로 노출하고, v1의 한계(차트 전용·생존 편향·미래참조 통제 범위)를 문서로 남긴다. 기존 규칙 기반 추천 엔진과 병존.

## Acceptance criteria

- [ ] (선택) 신규 신호 엔드포인트 — 기존 `routes.py` 관례 준수, 규칙 기반 추천과 **경로 분리**
- [ ] 응답에 신호/확률/SHAP 근거 포함
- [ ] 한계 문서화: 차트 전용(재무·뉴스 v2), 생존 편향 잔존, 5거래일 단일 지평
- [ ] 기존 규칙 기반 `recommendation_engine.py` 미변경 확인 (병존)
- [ ] (선택) ADR 작성: `0007-ml-신호-지도학습-도입.md` — 왜 규칙 기반이 아니라 학습 기반인가
